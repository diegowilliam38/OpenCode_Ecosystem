# API_SPEC — Contratos dos artefatos do projeto

## Artefato: Anteprojeto PDF (T0.6)
### Requisitos
- Formato: PDF/A
- Laudas: ≤ 7 (excluindo referências)
- Margens: superior/inferior 2,5cm; esquerda/direita 3,0cm
- Fonte: Times New Roman 12pt
- Espaçamento: 1,5
- Anônimo: zero marcas de autoria no corpo do texto
- Tamanho: componente do arquivo único ≤ 15MB

### Estrutura obrigatória
1. Título + Linha de Pesquisa + Tema
2. Resumo (até 250 palavras) + 3-5 palavras-chave
3. Justificativa e delimitação do problema
4. Objetivos (geral + específicos)
5. Fundamentação teórica (3 eixos)
6. Metodologia (3 fases)
7. Cronograma
8. Referências (ABNT NBR 6023:2018)

### Validação
- [ ] ≤ 7 laudas
- [ ] Anônimo
- [ ] Margens ABNT
- [ ] Todas as referências com DOI verificável
- [ ] Zero travessões
- [ ] Siglas expandidas na primeira ocorrência

## Artefato: Arquivo Único de Inscrição (T0.11)
### Ordem exata (Edital nº 01/2026, item 3.6.1)
1. RG/CNH (frente e verso) com CPF visível
2. Diploma de Graduação (frente e verso)
3. Currículo Lattes atualizado (PDF)
4. Ficha de Produção Acadêmica + TODAS as comprovações
5. Autodeclaração (Anexo II) — se cotista
6. Declaração de uso de IA (Anexo IV)
7. Anteprojeto de Pesquisa

### Validação
- [ ] Arquivo único PDF
- [ ] ≤ 15MB
- [ ] Ordem exata do edital
- [ ] Nome do arquivo: nome completo do candidato
- [ ] Enviado via SIGAA até 01/06/2026 23h59

## Artefato: Apresentação de Defesa (Fase 4)
### Requisitos
- Formato: PDF
- Slides: 10 (10 minutos de apresentação)
- Enviado por e-mail (ppgte@virtual.ufc.br) até 24h antes
- Assunto: "Arquivo de Apresentação para Defesa de Anteprojeto"
- Conter foto recente e nítida do candidato

### Estrutura
1. Capa (título, linha, tema, orientador, foto)
2. Problema: "caixa-preta" das IAs + alucinações + LGPD
3. Solução: OpenCode Ecosystem (arquitetura, prints)
4. Objetivos (geral + específicos)
5. Eixo 1: IA Multiagente (Russell & Norvig, Wooldridge)
6. Eixo 2: Tecnologia Educacional (Valente, Moran, Siemens)
7. Eixo 3: Ética e LGPD (Floridi, Brasil 2018)
8. Metodologia: 3 fases + cronograma
9. Produto educacional: guia prático digital
10. Considerações finais + agradecimento

## Artefato: Guia Prático Digital (Fase 2)
### Requisitos
- Web responsivo (HTML/CSS/JS)
- 4 módulos independentes
- Cada módulo com: objetivo, passo a passo, exemplos, checklist
- Navegação por teclado (← →)
- Funciona offline
